---

## Overview

Amygdala is a real-time survival decision engine for travelers. The backend aggregates 6+ data sources, computes trust scores, and returns **ONE deterministic instruction**: SHELTER, STAY, MOVE, EVACUATE, or MONITOR.

Your job as the frontend developer: build the mobile app (React Native / Flutter / PWA) that consumes this API. The current web UI at `/` is a testing reference — replace it with a proper native experience.

---

## Architecture

```
User's Phone (your app)
    │
    ├── GET  /api/v1/decision        ← THE primary call (every 60s)
    ├── GET  /api/v1/alerts/...      ← Threat list for UI
    ├── GET  /api/v1/transport/...   ← Nearby transport
    ├── POST /api/v1/reports         ← User incident reports
    ├── GET  /api/v1/offline/pack/.. ← Pre-download for offline
    └── POST /api/v1/simulate/...    ← Demo/testing only
```

**Data sources (backend handles all of this):**
- USGS (earthquakes) + GDACS (disasters) + NASA EONET (wildfires, storms)
- UK FCDO (geopolitical travel advisories) + Meteoalarm (EU weather)
- ReliefWeb / UN OCHA (humanitarian crises) + OpenStreetMap (transport)

## Primary Endpoint: Decision Engine

### `POST /api/v1/decision`

**This is the only endpoint you MUST call.** Everything else is supplementary.

#### Request
```json
{
  "latitude": 42.01,
  "longitude": 23.10,
  "connectivity_status": "online",
  "language": "en"
}
```

#### Response
```json
{
  "instruction": "NO IMMEDIATE THREAT — stay aware of your surroundings.",
  "action": "MONITOR",
  "urgency": "LOW",
  "phase": "EVALUATE",
  "confidence": 0.50,
  "fallback_instruction": "Continue monitoring. Conditions can change rapidly.",
  "fallback_action": "MONITOR",
  "threat_summary": "",
  "local_emergency_number": "112",
  "nearest_safe_option": "Благоевград (train station, 1.3km)",
  "trust_score": 0.00,
  "sources_agreeing": 0,
  "sources_total": 0,
  "location": { "latitude": 42.01, "longitude": 23.10, ... },
  "advisory_meta": {
    "disclaimer": "IMPORTANT: This information is generated by...",
    "medical_disclaimer": "...",
    "data_freshness_warning": "...",
    "generated_at": "2026-03-28T11:00:00Z",
    "data_sources": [...],
    "ai_model": "deterministic-rule-engine-v1",
    "confidence": 0.50
  },
  "decided_at": "2026-03-28T11:00:00Z"
}
```

#### Key Fields for the UI

| Field | Type | UI Usage |
|-------|------|----------|
| `instruction` | string | **THE big text.** Display prominently. User reads this in ≤2 seconds. |
| `action` | enum | `SHELTER` / `STAY` / `MOVE` / `EVACUATE` / `MONITOR` — use for color coding |
| `urgency` | enum | `HIGH` (red) / `MEDIUM` (orange) / `LOW` (green) |
| `phase` | enum | `SURVIVE` / `STABILIZE` / `EVALUATE` / `ESCAPE` — show as progress bar |
| `confidence` | 0-1 | Show as percentage. Below 0.3 = "limited data" |
| `fallback_instruction` | string | Show smaller, below main instruction |
| `local_emergency_number` | string | The emergency call button uses this |
| `nearest_safe_option` | string | Show when action is MOVE/EVACUATE |
| `trust_score` | 0-1 | Multi-source agreement. Higher = more reliable |
| `sources_agreeing` / `sources_total` | int | "3 of 4 sources agree" |

#### Action → Color Mapping
```
SHELTER  → #EF4444 (red)      — immediate danger
STAY     → #F59E0B (orange)   — dangerous to move
MOVE     → #F59E0B (orange)   — get to exit
EVACUATE → #EF4444 (red)      — leave the area
MONITOR  → #22C55E (green)    — safe, stay aware
```

#### Phase → Progress Bar
```
SURVIVE   → step 1/4 (red)    — 0-5 minutes
STABILIZE → step 2/4 (orange) — 5-60 minutes
EVALUATE  → step 3/4 (yellow) — 1-24 hours
ESCAPE    → step 4/4 (green)  — 24h+ (safe to move)
```

## Supplementary Endpoints

### `GET /api/v1/alerts/region/{lat}/{lon}?radius_km=500`

Returns all active alerts near the user. Use to populate a threat list.

```json
{
  "alerts": [
    {
      "id": "usgs-abc123",
      "type": "earthquake",
      "severity": "high",
      "title": "M5.5 Earthquake — 1 km NW of Rodotópi, Greece",
      "description": "...",
      "issued_at": "2026-03-28T10:00:00Z",
      "location": { "latitude": 39.5, "longitude": 20.8 },
      "radius_km": 324.5,
      "source": { "name": "USGS Earthquake Hazards Program", "reliability": "official" },
      "official_url": "https://earthquake.usgs.gov/..."
    }
  ],
  "location": { ... },
  "retrieved_at": "...",
  "sources": [...]
}
```

Severity values: `critical` / `high` / `medium` / `low` / `info`
Alert types: `earthquake` / `flood` / `wildfire` / `hurricane` / `tornado` / `tsunami` / `volcanic_eruption` / `geopolitical` / `civil_unrest` / `terrorism` / `pandemic` / `industrial_accident` / `landslide` / `other`

### `GET /api/v1/transport/status/{lat}/{lon}`

Real transport infrastructure near the user (from OpenStreetMap).

```json
{
  "options": [
    {
      "id": "osm-12345",
      "type": "train_station",
      "name": "Благоевград",
      "location": { "latitude": 42.02, "longitude": 23.09 },
      "status": "unknown",
      "distance_km": 1.3,
      "estimated_travel_minutes": 4,
      "source": { "name": "OpenStreetMap", "reliability": "verified" }
    }
  ]
}
```

Transport types: `airport` / `train_station` / `bus_station` / `port`
Status: `operational` / `disrupted` / `closed` / `unknown`

---

### `POST /api/v1/reports`

Submit a user incident report. Rate-limited to 5 per device per hour.

#### Request
```json
{
  "latitude": 42.01,
  "longitude": 23.10,
  "report_type": "earthquake",
  "description": "Felt shaking for 3 seconds",
  "severity_estimate": "medium",
  "device_id": "unique-device-identifier"
}
```

`report_type`: `earthquake` / `flood` / `fire` / `conflict` / `infrastructure` / `other`
`severity_estimate`: `low` / `medium` / `high` / `critical` / `unknown`

#### Response
```json
{
  "report_id": "abc123",
  "accepted": true,
  "trust_contribution": 0.1,
  "reports_in_area": 3,
  "submitted_at": "2026-03-28T11:00:00Z"
}
```

### `GET /api/v1/offline/pack/{country_code}`

Download an offline data pack before losing internet. Country codes: `BG`, `TR`, `GR`, `DE`, `JP`, `IL`, `UA`, etc.

```json
{
  "country_code": "BG",
  "country_name": "Bulgaria",
  "emergency": {
    "emergency_number": "112",
    "police": "166",
    "ambulance": "150",
    "fire": "160",
    "language_tips": ["'Pomosht' = Help", "'Bolnitsa' = Hospital"]
  },
  "offline_rules": [
    "IF you feel the ground shake: DROP, COVER, HOLD ON...",
    ...
  ],
  "generated_at": "...",
  "valid_until": "..."
}
```

**Store this in local storage / SQLite.** When offline, the app should use these rules instead of calling the API.

## Simulation Endpoints (Demo Only)

### `GET /api/v1/simulate/scenarios`
Lists available simulation scenarios.

### `POST /api/v1/simulate/{scenario}?lat=42.01&lon=23.10`
Runs a simulation through the real decision engine with fake data.

Available scenarios:
- `earthquake_nearby` — M6.2 earthquake 15km away. Triggers SHELTER.
- `geopolitical_crisis` — Armed conflict, FCDO advises against all travel. Triggers EVACUATE.
- `flood_warning` — Flash flooding, river overflow. Triggers MOVE.
- `all_clear` — No threats. Triggers MONITOR.

## Recommended App Flow

### 1. Startup
```
GET /health → verify API is available
GET /offline/pack/{country} → cache for offline use
Request GPS permission
Request notification permission
```

### 2. Main Loop (every 60 seconds)
```
POST /decision (GPS coordinates) → update the main instruction
GET /alerts/region/{lat}/{lon}    → update threat list
```

### 3. User Interaction
```
POST /reports → user submits an incident report
GET /transport/status/{lat}/{lon} → show transport options
```

### 4. Offline Mode
```
If API call fails → use cached offline pack
Show offline rules from the pack
Display last known decision
Show "OFFLINE" indicator
```

## Disclaimer Handling

**Every response from `/decision` and `/emergency/bundle` includes `advisory_meta.disclaimer`.** You MUST display this somewhere in the UI (can be a small footer or expandable section). This is a legal requirement.

---

## CORS

The API allows all origins (`*`). You can call it from any domain or app.

---

## Data Freshness

The backend fetches from 6+ sources **on every request** — data is always real-time. There is no caching layer between the user and the data sources. The `advisory_meta.generated_at` timestamp shows exactly when the data was assembled.
